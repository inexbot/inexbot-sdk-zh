# 纳博特二次开发文档库

# 文档总览

本文档库面向使用纳博特（NEXDROID）机器人控制系统的二次开发用户，包含完整的 API 参考、开发指南和示例代码。

---

# 1. 开发形式概览

纳博特系统提供多种二次开发层级，从底层到高层依次为：

| 开发层级 | 说明 | 适用场景 | 开发语言/环境 |
|----------|------|----------|---------------|
| **控制器二次开发** | 直接在控制器内运行程序 | 专有工艺开发、特殊机型支持、插补算法替换 | Linux, C++ |
| **示教器二次开发** | 定制专属UI界面和功能 | UI/UE定制、专有功能界面 | Linux, Qt/C++ |
| **上位机SDK** | 通过API直接控制机器人 | 摆脱示教器束缚、远程控制 | Windows/Linux, C++/C#/Python |
| **JSON协议** | 基于JSON格式的TCP通讯 | 标准化协议集成、跨平台控制 | 任意语言 |
| **ROS开发** | ROS机器人操作系统集成 | 科研教育、ROS生态集成 | Linux, C++/Python |
| **HAL层开发** | 硬件抽象层，访问底层硬件 | 驱动开发、设备控制 | Linux, C++ |
| **EtherCAT主站** | 工业总线主站开发 | 实时控制、EtherCAT从站管理 | Linux, C++ |

# 3. 技术规格

## 3.1 控制器规格

| 项目 | 规格 |
|------|------|
| 机器人数量 | 支持最多4台机器人 |
| 坐标系类型 | 关节坐标、直角坐标、工具坐标、用户坐标、大地坐标 |
| 通讯端口 | 5000（文件传输）、6000（命令控制）、7000（状态广播） |
| 外部轴 | 支持最多5个外部轴 |

## 3.2 开发环境

| 组件 | 要求 |
|------|------|
| 操作系统 | Ubuntu 20.04 LTS（推荐） |
| 编译器 | GCC 9.4.0 或更高 |
| 架构 | x86_64, ARM, ARM64 |

# 4. 推荐学习路径

## 4.1 上位机控制（推荐入门）

适合：需要快速集成机器人控制的工程师

1. 阅读 **02-入门指南.md** 选择开发形式
2. 学习 **05-上位机SDK/** 中对应语言的开发指南
3. 参考示例代码进行实践

## 4.2 工艺定制开发

适合：需要定制焊接、码垛等工艺的用户

1. 掌握上位机基础控制
2. 学习 **08-JSON协议参考/04-工艺应用/**
3. 参考控制器二次开发进行深度定制

## 4.3 科研与ROS集成

适合：科研用户、ROS开发者

1. 学习 **07-ROS开发/** 中的安装教程
2. 参考ROS示例进行机器人控制
3. 结合JSON协议进行高级功能开发

# 5. 术语表

| 术语 | 说明 |
|------|------|
| **NexDroid** | 纳博特机器人控制软件平台 |
| **DH参数** | Denavit-Hartenberg参数，机器人运动学模型参数 |
| **Palletizing** | 码垛工艺 |
| **EtherCAT** | 一种工业以太网现场总线 |
| **PDO** | Process Data Object，EtherCAT过程数据对象 |
| **SDO** | Service Data Object，EtherCAT服务数据对象 |
| **ROS** | Robot Operating System，机器人操作系统 |
| **工具手** | Tool Center Point (TCP)，工具中心点 |
| **用户坐标** | User Coordinate System (UCS)，用户自定义坐标系 |
