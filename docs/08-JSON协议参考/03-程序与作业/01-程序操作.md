# 01-程序操作

## 1. 程序打开（0x5100）

打开程序后才能执行作业文件。

```json
{
  "robot": 1,
  "jobName": "myprogram",
  "suffixname": ".JBR"
}
```

- `robot`: 机器人号[1,4]
- `jobName`: 作业文件名
- `suffixname`: 后缀名（.JBR前台/.JBP后台局部/.JBPG后台全局）

## 2. 程序复制（0x5101）

复制现有程序生成新文件。

```json
{
  "robot": 1,
  "oldjobName": "source_program",
  "newjobName": "target_program",
  "suffixname": ".JBR"
}
```

## 3. 程序清除（0x5102）

清空当前机器人所有程序（不包括复位程序）。

```json
{
  "robot": 1,
  "lock_enable": true
}
```

- `lock_enable`: 是否清空作业文件使能

## 4. 获取当前打开程序（0x5103 → 0x5104）

查询当前已打开的作业文件。

```json
// 请求
{"robot": 1}
// 响应
{"openedJobName": "myprogram"}
```

## 5. 主程序设置（0x5110/0x5111/0x5112）

主程序是特殊的前台程序，只有前台程序可设置为主程序。

### 设置主程序（0x5110）

```json
{
  "robot": 1,
  "jobName": "main_program"
}
```

- 空字符串表示取消主程序

### 查询主程序（0x5111 → 0x5112）

```json
// 请求
{"robot": 1}
// 响应
{"robot": 1, "jobName": "main_program"}
```

## 6. 程序保存（0x5120）

修改、插入、删除指令后需要保存。

```json
{
  "robot": 1,
  "jobname": "myprogram",
  "suffix": ".JBR",
  "time": "2024.08.08 14:16:39"
}
```

## 7. 发送程序指令（0x5122）

无作业文件时的指令插入，运行方式与作业运行相同，需设置`noFileRun=true`。

```json
{
  "robot": 1,
  "cmd": {...}
}
```

- `cmd`: 指令详情（见指令系统）

## 8. 程序同步（0x1222/0x1223/0x1224/0x1225）

### 上位机通知控制器同步（0x1222 → 0x1223）

```json
// 请求：无
// 响应
{"finished": true}
```

### 控制器通知示教器同步（0x1224 → 0x1225）

```json
// 请求：无
// 响应
{"finished": true}
```

## 9. 同步程序打开（0x3130）

控制器通知上位机程序已打开。

```json
{
  "robot": 1,
  "jobName": "opened_program"
}
```
