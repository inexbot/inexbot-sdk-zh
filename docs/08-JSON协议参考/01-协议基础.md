# JSON 通讯协议

# 1. 协议概述

纳博特控制器采用 JSON over TCP/IP 通讯协议，实现与示教器、上位机等外部设备的数据交互。

## 1.1 数据包格式

采用标准 TLV（Type-Length-Value）格式，包含固定包头和 CRC 校验。

| 字段 | 长度 | 说明 |
|------|------|------|
| SyncByte | 2 Byte | 固定包头 `0x4E66` |
| Length | 2 Byte | 16进制长度 0-2000，不含数据头、命令字、CRC |
| Command | 2 Byte | 命令字 |
| Data | Length | JSON 数据段 |
| CRC | 4 Byte | CRC32 校验（不含包头） |

## 1.2 数据示例

设置机器人1伺服状态为0（停止）：

```
发送：4e 66 00 17 20 01 7b 22 72 6f 62 6f 74 22 3a 31 2c 22 73 74 61 74 75 73 22 3a 30 7d 0a 6b 92 6d ff
```

分解说明：

| 字段 | 值 | 含义 |
|------|-----|------|
| `4e 66` | 0x4E66 | 固定包头 |
| `00 17` | 23 | 数据段长度 23 字节 |
| `20 01` | 0x2001 | 命令字 |
| `7b...7d 0a` | `{"robot":1,"status":0}\n` | JSON 数据 |
| `6b 92 6d ff` | CRC32 | 校验码 |

## 1.3 控制器端口

纳博特控制器使用三个主要通讯端口：

| 端口 | 协议 | 用途 |
|------|------|------|
| **5000** | TCP | 文件传输（作业文件、配置文件导入导出）、软件升级。大文件传输可使用 FTP。 |
| **6000** | TCP | 常规命令传输，示教器编程开发。采用心跳机制保持连接。 |
| **7000** | TCP/UDP | 状态广播信息，无需心跳机制。实时监控机器人状态。 |

## 1.4 心跳检查机制

6000 端口建立连接后需定期发送心跳包，确保通讯正常。

**机制说明：**

1. 客户端定期发送心跳检查包给控制器
2. 控制器收到后回复心跳回应包（携带时间戳）
3. 客户端检查回应数据是否正确
4. 超时时间：**6 秒**

**心跳超时处理：** 数据不正确或超时未收到回应，判定为连接中断。

# 2. 命令字体系

## 2.1 命令字编码规则

协议命令字采用 16 进制编码，结构如下：

| 前缀 | 范围 | 类别 |
|------|------|------|
| 1xxx | 1xxx, 10xx-1Fxx | 系统类 |
| 2xxx, 3xxx, 4xxx | 2000-4FFF | 机器人控制类 |
| 5xxx | 5000-5FFF | 程序与作业文件类 |
| 6xxx | 6000-6FFF | 变量类 |
| 7xxx | 7000-7FFF | 外设通讯类 |

## 2.2 系统类命令（1xxx）

| 命令字 | 功能 |
|--------|------|
| 10xx | 系统基础操作 |
| 11xx | 报错相关 |
| 12xx | 操作参数 |
| 13xx | 示教器与上位机通讯连接 |
| 1Dxx | 5000 端口协议 |
| 1Exx | 7000 端口协议 |

## 2.3 机器人控制类命令（2xxx-4xxx）

| 命令字范围 | 功能类别 |
|------------|----------|
| 2xxx | 参数设置（DH参数、关节参数、伺服参数等） |
| 30xx-34xx | 运动控制、伺服设置、状态 |
| 35xx | 工具手、用户坐标系、大地坐标系 |
| 36xx | 零点、复位点 |
| 37xx | 标定（四点标定、20点标定、外部轴标定） |
| 38xx | 干涉区、跟随误差、电机过载保护 |
| 4xxx | 动力学、力学功能、拖动示教、遥操作 |

## 2.4 程序与作业类命令（5xxx）

| 命令字范围 | 功能 |
|------------|------|
| 51xx-52xx | 程序增删改查 |
| 53xx | 程序导入导出 |
| 54xx | 指令操作 |
| 55xx | 程序运行控制（启动、暂停、停止） |
| 56xx | 程序状态 |
| 57xx | 远程程序 |
| 58xx | 程序自启动 |

## 2.5 外设通讯类命令（7xxx）

| 命令字范围 | 功能 |
|------------|------|
| 70xx | 总线（EtherCAT、CANopen） |
| 71xx | IO 配置、状态、功能 |
| 72xx | 其他外设 |
| 73xx | Modbus |
| 74xx | TCP 通讯 |
| 75xx | 数据上传 |
| 76xx | FinsTCP |
| 77xx | OPC UA |

# 3. 坐标类型说明

协议中使用数字代码表示不同坐标系：

| 值 | 坐标系类型 | 说明 |
|----|------------|------|
| 0 | 关节坐标 | 各关节角度值 |
| 1 | 直角坐标 | X/Y/Z + Rx/Ry/Rz |
| 2 | 工具坐标 | 相对于工具末端 |
| 3 | 用户坐标 | 用户自定义坐标系 |
| 4 | 电机坐标 | 电机编码器值 |

---

# 4. 机器人编号

纳博特控制器支持多机器人模式：

| 编号 | 说明 |
|------|------|
| 1-4 | 机器人号 1-4 |
| 默认为 1 | 单机模式时使用 |

---

# 5. 数据类型约定

| 类型 | 说明 | 示例 |
|------|------|------|
| int | 整数 | `"robot":1` |
| double | 浮点数 | `"pos":[0.0,0.0,0.0...]` |
| bool | 布尔值 | `"result":true` |
| string | 字符串 | `"name":"test"` |
| array | 数组 | `"vel":[0,0,0,0,0,0,0]` |
| object | JSON对象 | `{"robot":1,"status":0}` |
