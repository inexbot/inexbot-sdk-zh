# 7000 端口通讯(2207)

控制器为服务端，默认端口号为7000

本协议所规定的所有数据包均符合本格式定义，采用标准的TLV（Type,Length,Value）格式,

有固定起始包头和CRC校验。

| 项目 | 长度(Byte) | 说明 |
| SyncByte | 2 | 0x4E66固定包头 |
| Length | 2 | 16进制长度0-0xffff;不包含数据头和命令字，不包含CRC段 |
| Command | 2 | 命令字 |
| data | Length | 数据段（主要为JSON数据格式） |
| CRC | 4 | 对不含包头数据做CRC校验(CRC32) |

配置文件设置：

在配置文件controller.json中，”hostComputerService”节点下：

”socketType”节点表示socket类型，可选择”TCP”和”UDP”，默认为”TCP”；

”checkCRC”节点表示是否进行CRC检测，选择false时，不检测CRC，但注意CRC所占用的4个字节的数据仍要发送；默认为true，检测CRC。

#### 查询状态  命令字  0x9512

{

“channel“:1, //最多9个通道

“stop”:1, //为1时,停止之前的持续发送,下面的数据可不发

“robot”:1,

“mode”:0,  //0：应答(请求一次回复一次)，1：持续(请求一次持续回复)

“interval”:10, //ms，仅mode为持续时有效，范围10-60000

“queryType”:  //无需查询的不写即可

[ “realPosACS”,“realPosMCS”,“realPosPCS”,“realPosUCS”, //实时坐标位置，关节,直角,工具,用户

“axisVel”, “axisAcc”,  //轴速度，轴加速度

“torque”, “electric”,  //力矩，电流

“numGVar”,  //数值变量

“IO”,  //IO端口

“serialNum”,//序列号

"runControlPosRunNum",//运动控制当前轨迹已运行了几个点位

”detailedMotionPos”  //详细运动点位，该项每个机器人同一时刻仅支持一个通道查询

],

“typeCfg”:{  //查询内容配置，仅需要查询相关类型数据时有效

“numGVar”:[ “GD001”, “GA001”, “GI001”, “GI333” ],

“IO”:[ “DI1”, “DI16”, “DO1”, “DO3”, “DO17”],  //可查询的最大数量不应大于IO个数

“detailedMotionPos”:{

“num”:10, //每帧数据回复的点位数目

“optional”: //可选参数

[“ACS”, ”MCS”, ”time”, ”reset”] //关节坐标，直角坐标，时间戳，重置点位记录

}

}

}

{

“success”:false,

“cause”:”timeout” //接收成功时为空，”busy”:当前有未传输完成的数据，”timeout”:接收超时，”dataErr”：数据错误，”termination”:发送端终止了正在传输的数据
