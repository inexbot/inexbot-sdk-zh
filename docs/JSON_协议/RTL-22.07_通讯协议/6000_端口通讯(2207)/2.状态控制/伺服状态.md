### 伺服状态设置及获取

#### 上位机改变伺服的状态，发送下面命令:

- int 类型
- 0：真实伺服，1：虚拟伺服，2：无伺服

- 命令字：0x2001 SERVO_STATUS_SET

- "robot": 1

- int 类型
- 选择机器人 1
- 选择机器人的取值范围 [1，4]

- "status":0

```
{
"robot":1,
"status":0
}
```
#### 上位机查询示教状态发送给控制器的命令:

- int 类型
- 0：停止，1：就绪，2：错误，3：运行

- 命令字：0x2002 SERVO_STATUS_INQUIRE

- "robot": 1

```
{
"robot":1
}
```
#### 控制器收到查询命令，或者当前伺服状态发生改变时，回复下面命令：

*0x2003 SERVO_STATUS_RESPOND

- int 类型
- 选择机器人 1
- 选择机器人的取值范围 [1，4]

- 命令字：0x2002 SERVO_STATUS_INQUIRE

- "robot": 1

- int 类型
- 选择机器人 1
- 选择机器人的取值范围 [1，4]

- "mode": 0

- int 类型
- 0：单机模式，  1：多机模式

- "status":0

```
{
"mode":0,
"robot":1,
"status":0
}
```

- 0：DEADMAN下电，1：DEADMAN上电

- 命令字：0x2304 DEADMAN_MODE_SET

- int 类型
- deadmanMode为1时，表示上使能端口使能IO端口号，每个端口最多可分配16个IO序号

- "deadmanPortTwo":0

```
{
"deadmanMode"：1,
"deadmanPortOne":9,
"deadmanPortTwo":0
}
```

### 清除伺服错误

#### 示教器清除伺服错误的时候，发送下面命令：

*0x3201 FAULT_RESET

{

"robot":1,

}

#### 清除伺服错误后，返回清除结果：

*0x3202 FAULT_RESET_RESULT

{

"robot":1,

"clearErrflag":true,        //true：清除伺服错误成功，false：清除失败

}

- int 类型
- 选择机器人 1
- 选择机器人的取值范围 [1，4]
