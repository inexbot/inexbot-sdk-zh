# Python 开发指南

## 版本兼容性说明

不同版本的 Python 调用 windows下的 .py 或 linux下的 .so 文件（Python 的动态链接库文件）会存在兼容性问题，原因如下：

- **Python ABI（应用二进制接口）不同**：每个 Python 版本的 ABI 可能不同，即使在小版本之间（例如 3.10 和 3.11）也可能有所变化。这会导致低版本的 Python 无法直接使用高版本编译的 .pyd 文件。
- **编译器版本的差异**：.pyd 文件需要使用特定的编译器编译，而不同 Python 版本通常有推荐的编译器版本。例如，Windows 上的 Python 3.8 和 3.9 推荐的 GCC 版本不同，所以即使代码相同，不同编译器编译的 .pyd 文件在加载时也可能不兼容。
- **依赖的第三方库版本**：如果 .pyd 文件依赖其他动态链接库（如 C/C++ 库），不同 Python 版本的链接规则可能有所变化，导致兼容性问题。
- **Python 标准库的变化**：不同 Python 版本对 C API 的支持可能会有所不同，某些 API 在高版本可能发生更改或被移除。这些变化会导致低版本的 Python 无法加载高版本编译的 .pyd 文件。

**因此，我们要求您使用的 Python 版本与我们编译windows下的 .pyd 或linux下的 .so 文件的 Python 版本强一致，以确保兼容性。**
