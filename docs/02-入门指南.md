# 入门指南

本文档帮助你根据需求选择合适的二次开发形式，并了解相关技术要求。

---

# 1. 开发形式总览

| 开发形式 | 应用场景 | 操作系统 | 开发语言 |
|----------|----------|----------|----------|
| **控制器二次开发** | 专有工艺开发、特殊机型支持、插补算法替换 | Linux | C++ |
| **示教器二次开发** | 定制专属 UI/UE 和功能 | Linux | Qt / C++ |
| **上位机开发** | 直接通过 API 控制机器人，摆脱示教器限制 | Windows / Linux | C++ / C# / Python / Qt |
| **主站开发** | 提供实时系统和 EtherCAT 主站功能 | Linux | C / C++ |
| **JSON 协议开发** | 遵循 JSON 协议进行远程控制和数据交换 | Windows / Linux | 任意语言 |
| **ROS 开发** | 基于 ROS 协议控制和集成纳博特控制器 | Linux | C++ / Python |

---

# 2. 各类开发形式详解

## 2.1 控制器二次开发

**适用场景：** 专有工艺开发、特殊机型支持、插补算法替换等

**必须掌握的知识：**
- 操作系统：Linux 系统
- 编程语言：C++
- 主要功能：开发控制器底层算法和特定硬件的支持

**开发内容：**
- 替换插补算法
- 定制硬件接口
- 调整路径规划
- 实时控制优化

**适合用户：** 控制器硬件和算法开发人员

---

## 2.2 示教器二次开发

**适用场景：** 定制专属用户界面（UI/UE）和功能

**必须掌握的知识：**
- 操作系统：Linux 系统
- 编程语言：Qt / C++
- 主要功能：定制化的用户界面设计，结合控制器实现专有功能

**开发内容：**
- 使用 Qt 框架开发图形界面
- 设计控制面板、手动操作界面
- 配合控制器定制动作和命令输入

**适合用户：** UI/UE 开发人员

---

## 2.3 上位机开发

**适用场景：** 直接通过 API 控制机器人，摆脱示教器限制

**必须掌握的知识：**
- 操作系统：Windows（VC++/C#/VB/Python/Qt）或 Linux（C++/Qt/Python）
- 主要功能：通过 API 直接控制机器人，进行高级控制和监控

**开发内容：**
- 通过网络接口与控制器交互
- 发送指令并获取反馈信息
- 复杂的远程控制和数据处理

**适合用户：** 需要进行机器人高层控制、自动化操作或系统集成的开发人员

---

## 2.4 主站开发

**适用场景：** 将控制器退化为工控机，提供实时系统和 EtherCAT 主站功能

**必须掌握的知识：**
- 操作系统：Linux 系统
- 编程语言：C / C++
- 主要功能：实现 EtherCAT 主站与设备的通信

**开发内容：**
- 编写实时任务
- 开发 EtherCAT 主站协议
- 与控制器和其他设备的通信

**适合用户：** 工业自动化和嵌入式系统开发人员

---

## 2.5 JSON 协议开发

**适用场景：** 通过遵循 JSON 协议与纳博特控制器进行通信

**必须掌握的知识：**
- 操作系统：Windows（VC++/C#/VB/Python/Qt）或 Linux（C++/Qt/Python）
- 主要功能：基于 JSON 协议开发通信接口

**开发内容：**
- 通过 JSON 格式消息与机器人控制器交互
- 发送控制指令和获取执行结果

**适合用户：** 需要标准化协议与机器人集成的开发人员

---

## 2.6 ROS 开发

**适用场景：** 基于 ROS 协议开发和控制纳博特控制器

**必须掌握的知识：**
- 操作系统：Linux 系统
- 编程语言：C++ / Python
- 主要功能：通过 ROS 控制纳博特控制器，集成到 ROS 网络中

**开发内容：**
- 开发 ROS 节点
- 与其他 ROS 节点协同工作
- 机器人控制和数据处理

**适合用户：** ROS 开发人员，尤其是熟悉 ROS 环境并希望集成纳博特控制器的工程师

---

# 3. 受众推荐

| 目标受众 | 推荐开发方式 | 开发侧重点 |
|----------|-------------|-----------|
| **本体厂商** | 控制器二次开发、示教器二次开发 | 深度定制控制系统和用户界面 |
| **集成商** | 上位机开发、JSON 协议开发 | 高效通信和自有工艺保护 |
| **科研用户** | 控制器二次开发、ROS 开发 | 精确控制和复杂算法研究 |
| **教育用户** | 示教器二次开发、JSON 协议开发 | 易用性和跨平台数据传输 |

---

# 4. 推荐学习路径

## 4.1 上位机控制（推荐入门）

适合需要快速集成机器人控制的工程师：

1. 学习 **05-上位机SDK/** 中对应语言的开发指南
2. 参考示例代码进行实践
3. 逐步掌握运动控制、IO 操作等基础功能

## 4.2 工艺定制开发

适合需要定制焊接、码垛等工艺的用户：

1. 掌握上位机基础控制
2. 学习 **08-JSON协议参考/04-工艺应用/**
3. 参考控制器二次开发进行深度定制

## 4.3 科研与 ROS 集成

适合科研用户、ROS 开发者：

1. 学习 **07-ROS开发/** 中的安装教程
2. 参考 ROS 示例进行机器人控制
3. 结合 JSON 协议进行高级功能开发

---

# 5. 入门小贴士

1. **学习资料**：各开发方式都提供了丰富的文档和示例代码，建议先通读相关文档
2. **社区支持**：ROS 和 JSON 协议有广泛的社区支持
3. **灵活应用**：建议保持开发路径的开放性，有助于后期快速迭代
